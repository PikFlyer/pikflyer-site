First-Time Setup Guide首次連線教學

Follow these steps for your platform to get started.根據你的平台，按照以下步驟開始使用。

Prerequisites事前準備

  • macOS 13 (Ventura) or latermacOS 13 (Ventura) 或更新版本
  • Xcode Command Line Tools — open Terminal and run: xcode-select --installXcode 命令列工具 — 打開終端機執行：xcode-select --install
  • Python 3.9+ (comes with macOS or install via Homebrew)Python 3.9+（macOS 內建，或透過 Homebrew 安裝）
  • pymobiledevice3 — run: pip3 install pymobiledevice3pymobiledevice3 — 執行：pip3 install pymobiledevice3

1 Enable Developer Mode on iPhone開啟 iPhone 開發者模式

Go to Settings > Privacy & Security > Developer Mode. Toggle it on and restart your iPhone when prompted.前往 設定 > 隱私與安全性 > 開發者模式，開啟後依提示重新啟動 iPhone。

2 Start tunneld service啟動 tunneld 服務

Open Terminal and run: sudo pymobiledevice3 remote tunneld. Keep this window open. This creates a secure tunnel to your device.打開終端機執行：sudo pymobiledevice3 remote tunneld，保持視窗開啟。此服務建立與裝置的安全通道。

3 Connect via USBUSB 連接

Plug your iPhone into your Mac with a Lightning or USB-C cable. When prompted on your phone, tap "Trust".用 Lightning 或 USB-C 線將 iPhone 插上 Mac。手機彈出提示時點選「信任」。

⚠️ USB connection is required. WiFi mode is too slow for GPS simulation.⚠️ 必須使用 USB 連接。WiFi 模式太慢，不適合 GPS 模擬。

4 Launch Pik Flyer and connect啟動 Pik Flyer 並連接

Open Pik Flyer, click "Connect Device". Your iPhone should appear. Click it to start GPS simulation.開啟 Pik Flyer，點擊「連接裝置」，iPhone 會自動出現，點選即可開始 GPS 模擬。

Prerequisites事前準備

  • macOS 12 or latermacOS 12 或更新版本
  • Android Platform Tools (ADB) — install via Homebrew: brew install android-platform-toolsAndroid Platform Tools (ADB) — 透過 Homebrew 安裝：brew install android-platform-tools
  • A mock location app on your phone (e.g. "Fake GPS Location" from Play Store)手機上的模擬位置 App（例如 Play Store 上的「Fake GPS Location」）

1 Enable Developer Options開啟開發者選項

Go to Settings > About Phone, tap "Build Number" 7 times until you see "You are now a developer!"前往設定 > 關於手機，連續點擊「版本號碼」 7 次，直到看到「您現在是開發者了！」

2 Enable USB Debugging & Mock Location開啟 USB 偵錯 & 模擬位置

Go to Settings > Developer Options. Enable "USB Debugging". Then set "Select mock location app" to your chosen mock GPS app.前往設定 > 開發者選項，開啟「USB 偵錯」，並將「選取模擬位置應用程式」設為你安裝的 GPS 模擬 App。

3 Connect via USBUSB 連接

Plug your Android phone into your Mac with a USB cable. When prompted, allow "USB Debugging" and check "Always allow from this computer".用 USB 線將 Android 手機插上 Mac。彈出提示時允許「USB 偵錯」並勾選「一律允許這台電腦」。

4 Launch Pik Flyer and connect啟動 Pik Flyer 並連接

Open Pik Flyer, click "Connect Device". Your Android phone should appear. Select it and start exploring!開啟 Pik Flyer，點擊「連接裝置」，Android 手機會自動出現，選擇後即可開始探索！

Prerequisites事前準備

  • Windows 10/11 (64-bit)Windows 10/11（64 位元）
  • iTunes (Microsoft Store or Apple website) — required for Apple device driversiTunes（Microsoft Store 或 Apple 官網下載）— 必須安裝，用於 Apple 裝置驅動程式
  • Python 3.9+ — download from python.orgPython 3.9+ — 從 python.org 下載安裝
  • pymobiledevice3 — run in CMD: pip install pymobiledevice3pymobiledevice3 — 在 CMD 執行：pip install pymobiledevice3

1 Install iTunes first!先安裝 iTunes！

iTunes installs the Apple Mobile Device USB drivers that Windows needs to communicate with your iPhone. Without it, your phone won't be detected.iTunes 會安裝 Windows 所需的 Apple 行動裝置 USB 驅動程式。沒有它，電腦無法偵測到你的 iPhone。

⚠️ This is the most common issue on Windows. Install iTunes even if you don't plan to use it.⚠️ 這是 Windows 上最常見的問題。即使不使用 iTunes，也必須安裝。

2 Enable Developer Mode on iPhone開啟 iPhone 開發者模式

Go to Settings > Privacy & Security > Developer Mode. Toggle it on and restart.前往 設定 > 隱私與安全性 > 開發者模式，開啟後重新啟動。

3 Start tunneld service啟動 tunneld 服務

Open Command Prompt as Administrator and run: pymobiledevice3 remote tunneld. Keep this window open.以系統管理員身分打開命令提示字元，執行：pymobiledevice3 remote tunneld，保持視窗開啟。

4 Connect via USB and launchUSB 連接並啟動

Plug your iPhone in via USB, tap "Trust" on the phone, then open Pik Flyer and click "Connect Device".用 USB 線插上 iPhone，手機上點選「信任」，然後開啟 Pik Flyer 點擊「連接裝置」。

Prerequisites事前準備

  • Windows 10/11 (64-bit)Windows 10/11（64 位元）
  • Android Platform Tools (ADB) — download from developer.android.com/tools/releases/platform-tools, extract and add to PATHAndroid Platform Tools (ADB) — 從 developer.android.com/tools/releases/platform-tools 下載，解壓後加入 PATH
  • A mock location app on your phone (e.g. "Fake GPS Location" from Play Store)手機上的模擬位置 App（例如 Play Store 上的「Fake GPS Location」）

1 Enable Developer Options開啟開發者選項

Go to Settings > About Phone, tap "Build Number" 7 times.前往設定 > 關於手機，連續點擊「版本號碼」 7 次。

2 Enable USB Debugging & Mock Location開啟 USB 偵錯 & 模擬位置

In Developer Options, enable "USB Debugging" and set "Select mock location app" to your mock GPS app.在開發者選項中，開啟「USB 偵錯」，並將「選取模擬位置應用程式」設為 GPS 模擬 App。

3 Install OEM USB driver (if needed)安裝 OEM USB 驅動程式（如需要）

Some brands (Samsung, Xiaomi) require a specific USB driver. Samsung users: install Samsung USB Driver from Samsung's website. Others: try the universal Google USB Driver included in Platform Tools.部分品牌（三星、小米）需要安裝專屬 USB 驅動程式。三星用戶：從三星官網安裝 Samsung USB Driver。其他品牌：嘗試 Platform Tools 中的通用 Google USB Driver。

4 Connect via USB and launchUSB 連接並啟動

Plug your Android phone in, allow "USB Debugging" when prompted. Open Pik Flyer and click "Connect Device".插上 Android 手機，允許「USB 偵錯」，開啟 Pik Flyer 點擊「連接裝置」。

Frequently asked questions常見問題

Is Pik Flyer safe to use?Pik Flyer 安全嗎？

Pik Flyer simulates GPS at the OS level. It works with Pikmin Bloom and other location apps. Please use responsibly.Pik Flyer 在作業系統層級模擬 GPS，適用於 Pikmin Bloom 及其他定位 App。請負責任地使用。

What devices are supported?支援哪些裝置？

iPhone (iOS 17+) and Android (5.0+) via USB on Mac or Windows.iPhone (iOS 17+) 和 Android (5.0+)，透過 USB 連接 Mac 或 Windows。

Do I need to jailbreak?需要越獄嗎？

No. Uses Apple's Developer Tools protocol. No jailbreak needed.不用。使用 Apple 開發者工具協定，無需越獄。

How smooth is auto walk?自動走路順暢嗎？

Pik Flyer uses dual-thread architecture for continuous GPS signals — ultra-smooth, no jerky movement.Pik Flyer 使用雙線程架構連續發送 GPS 訊號，超平滑，不會走走停停。

Can I cancel anytime?可以隨時取消嗎？

Yes. Cancel anytime, keep access until billing period ends.可以，隨時取消，使用至帳單週期結束。

My iPhone is not detected — what should I do?我的 iPhone 偵測不到怎麼辦？

Make sure you: (1) have iTunes installed (Windows), (2) Developer Mode is on, (3) tunneld is running (sudo pymobiledevice3 remote tunneld), (4) you're using a USB cable (not WiFi), and (5) you tapped "Trust" on the phone.請確認：(1) 已安裝 iTunes（Windows）、(2) 開發者模式已開啟、(3) tunneld 正在執行（sudo pymobiledevice3 remote tunneld）、(4) 使用 USB 連接（非 WiFi）、(5) 手機上已點選「信任」。

My Android phone is not detected我的 Android 手機偵測不到

Ensure USB Debugging is enabled in Developer Options. When you plug in, select "File Transfer" mode (not "Charging Only"). Some brands need an OEM USB driver — Samsung users should install Samsung USB Driver from Samsung's website.確認已在開發者選項開啟 USB 偵錯。插入時選擇「檔案傳輸」模式（非「僅充電」）。部分品牌需安裝 OEM USB 驅動程式 — 三星用戶請從三星官網安裝 Samsung USB Driver。

Auto walk is stuttering / flowers are sparse自動走路卡頓 / 花很稀疏

This usually happens with WiFi connections. Always use a USB cable for smooth GPS. Also make sure tunneld is running — it enables the high-speed persistent GPS mode.這通常是因為使用 WiFi 連接。請務必使用 USB 線以確保 GPS 平滑。同時確認 tunneld 正在執行 — 它啟用高速持續 GPS 模式。

What is tunneld and why do I need it?什麼是 tunneld？為什麼需要它？

tunneld is a background service from pymobiledevice3 that creates a secure tunnel to your iPhone. It allows Pik Flyer to send GPS coordinates directly and instantly (every 0.3s) instead of through slow subprocess calls. This is what makes auto walk ultra-smooth.tunneld 是 pymobiledevice3 的背景服務，會建立與 iPhone 的安全通道。它讓 Pik Flyer 能直接即時發送 GPS 座標（每 0.3 秒），而非透過緩慢的子程序呼叫。這就是自動走路超平滑的關鍵。

Does GPS spoofing work over WiFi?WiFi 可以用 GPS 模擬嗎？

Technically yes, but it's much slower and causes jerky movement. We strongly recommend USB for the best experience, especially for flower planting.技術上可以，但速度會慢很多，走路會卡頓。我們強烈建議使用 USB 以獲得最佳體驗，特別是種花時。

Does it work with other games besides Pikmin Bloom?除了 Pikmin Bloom 還能用在其他遊戲嗎？

Pik Flyer simulates GPS at the OS level, so it works with any location-based app. However, it's specifically optimized for Pikmin Bloom's walking and flower-planting mechanics.Pik Flyer 在作業系統層級模擬 GPS，因此任何定位 App 都能使用。但我們特別針對 Pikmin Bloom 的散步和種花機制進行了優化。
